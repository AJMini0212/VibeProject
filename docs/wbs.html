Phase 1: 지도 & 검색 (MVP)

기간: 10-11주차 | 시간: 14시간

목표: 기본 검색과 브라우징 기능 완성

1.1 프로젝트 초기화 및 환경 구성 (3시간)

  • ✓ Flutter 프로젝트 생성 및 기본 구조 설계
  • ✓ Google Maps API 설정 (iOS/Android)
  • ✓ 개발 환경 테스트 (에뮬레이터/실제 기기)

1.2 Google Maps 기반 위치 서비스 (4시간)

  • □ Google Maps Widget 구현 (지도 표시, 줌/팬 제어)
  • □ GPS 위치 권한 요청 및 현재 위치 표시
  • □ 마커(Marker) 표시 및 클러스터링
  • □ 위치 서비스 오류 처리

1.3 Naver Place API 데이터 연동 (3시간)

  • □ Naver Search API 통합
  • □ 가게 정보 캐싱 (로컬 저장소)
  • □ 가게 리스트 모델 설계

1.4 검색 & 필터링 UI (2시간)

  • □ 검색 바(SearchBar) 구현
  • □ 필터 UI 구현 (거리, 평점, 카테고리)
  • □ 정렬 옵션 (거리순, 평점순, 최신순)

1.5 가게 상세 페이지 (2시간)

  • □ 상세 정보 화면 UI 구현
  • □ 지도 통합 (상세 페이지에 미니 맵)
  • □ 액션 버튼 (즐겨찾기, 길 찾기, 전화)

Phase 2: 리뷰 & 인증

기간: 12주차 + 13주차 | 시간: 18시간

목표: 사용자 시스템과 리뷰 커뮤니티 구현

2.1 Firebase 초기 설정 (2시간)

  • □ Firebase 프로젝트 생성 및 설정
  • □ Flutter에 Firebase 통합
  • □ Firebase 권한 및 보안 규칙 설정

2.2 사용자 인증 시스템 (4시간)

  • □ Firebase Authentication 통합
  • □ 로그인/회원가입 UI 구현
  • □ 세션 관리 (상태 유지)
  • □ 로그아웃 및 회원탈퇴

2.3 사용자 프로필 시스템 (3시간)

  • □ 사용자 프로필 모델 설계
  • □ Firestore에 사용자 정보 저장
  • □ 프로필 화면 UI 구현
  • □ 프로필 편집 기능

2.4 리뷰 작성/조회 시스템 (5시간)

  • □ 리뷰 데이터 모델 설계
  • □ 리뷰 작성 폼 UI
  • □ Firebase Storage에 이미지 저장
  • □ Firestore에 리뷰 저장
  • □ 리뷰 목록 조회 및 표시
  • □ 리뷰 수정/삭제 기능

2.5 평점 시스템 (2시간)

  • □ 별점 위젯 구현
  • □ 평점 평균 계산 및 표시
  • □ 평점 분포 표시

2.6 이미지 처리 및 캐싱 (2시간)

  • □ Cached Network Image 패키지 통합
  • □ 이미지 압축 및 최적화
  • □ 이미지 갤러리 UI 구현

Phase 3: 소셜 & 배포

기간: 13-15주차 | 시간: 20시간+

목표: 커뮤니티 기능 완성 및 앱 배포

3.1 팔로우/팔로워 시스템 (3시간)

  • □ 팔로우 관계 데이터 모델
  • □ 팔로우/언팔로우 버튼 구현
  • □ 팔로워/팔로잉 목록 화면
  • □ 상호 팔로우 감지

3.2 활동 피드 (4시간)

  • □ 피드 데이터 모델 설계
  • □ 피드 화면 구현
  • □ 피드 실시간 업데이트
  • □ 피드 성능 최적화

3.3 사용자 검색 (2시간)

  • □ 사용자 검색 기능
  • □ 사용자 검색 UI
  • □ 검색 결과에서 프로필 접근

3.4 리뷰 상호작용 (3시간)

  • □ 좋아요(Like) 기능
  • □ 댓글(Comment) 시스템
  • □ 좋아요/댓글 UI 구현
  • □ 알림 (선택사항)

3.5 클라우드 동기화 & 고급 필터 (2시간)

  • □ 즐겨찾기 클라우드 동기화
  • □ 고급 필터 구현
  • □ 저장된 필터 (선택사항)

3.6 성능 최적화 (3시간)

  • □ 이미지 최적화
  • □ 지도 성능 최적화
  • □ Firestore 쿼리 최적화
  • □ 렌더링 성능 개선

3.7 UI/UX 폴란드 (2시간)

  • □ 다크모드 지원
  • □ 로딩 상태 개선
  • □ 에러 처리 및 메시지
  • □ 애니메이션 추가

3.8 테스트 및 디버깅 (2시간)

  • □ 기능 테스트 (수동)
  • □ 기기별 테스트
  • □ 성능 프로파일링
  • □ 보안 검토

3.9 앱 배포 준비 (2시간)

  • □ Google Play Console 설정
  • □ Apple App Store 설정
  • □ 앱 서명 및 빌드
  • □ 스토어 리스팅 최적화

3.10 배포 및 모니터링 (2시간)

  • □ Google Play 배포
  • □ App Store 배포
  • □ 배포 후 모니터링
  • □ 긴급 패치 준비

공통: 프로젝트 관리 & 문서화

기간: 전체 | 시간: 8시간

4.1 프로젝트 기획 및 계획 수립 (2시간)

  • ✓ 프로젝트 요구사항 분석
  • ✓ WBS, 일정 수립
  • □ 위험 관리 계획

4.2 기술 문서 작성 (2시간)

  • □ 개발 가이드 (DEVELOPMENT.md)
  • □ API 문서 (API.md)
  • □ 코드 스타일 가이드

4.3 중간/최종 발표 준비 (2시간)

  • □ 12주차 중간 발표 준비
  • □ 15주차 최종 발표 준비

4.4 버전 관리 & 협업 (2시간)

  • ✓ Git 저장소 관리
  • □ 변경 로그 관리 (CHANGELOG.md)
  • □ 릴리스 노트 작성

📈시간 분배 요약

Phase 2 리뷰/인증 18시간 24%
Phase 3 소셜/배포 20시간+ 27%
공통 관리/문서 8시간 11%
예비 시간 ~8시간 ~10%
합계 ~68시간 100%

🔗의존성 관계

Phase 1 → Phase 2 → Phase 3

  • Phase 1의 70% 이상 완료 후 Phase 2 시작 가능
  • Phase 2는 Phase 1과 부분적으로 동시 진행 가능 (마지막 주에 통합)
  • Phase 3는 Phase 1, 2 완료 후 시작
  • 배포는 Phase 3 + 테스트 & 최적화 완료 필요

✅품질 게이트 (Quality Gates)

마일스톤 기준
Phase 1 완료 지도에서 가게 50개+ 표시, 검색 기능 정상
Phase 2 완료 리뷰 5개+, 계정 생성 및 로그인 정상
Phase 3 완료 팔로우 기능 정상, 피드 실시간 업데이트
배포 준비 크래시 0, 성능 지표 만족
배포 완료 스토어 출시, 모니터링 설정

범례: ✓ = 완료 | □ = 미완료 | ⚠️ = 진행 중

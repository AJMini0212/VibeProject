Phase 1: 지도 & 검색 + 상세 페이지 (MVP) ✓ 완료

기간: 10-11주차 | 시간: 14시간 | 상태: 100% 완료

목표: 기본 검색과 브라우징 기능 완성 (완료)

1.1 프로젝트 초기화 및 환경 구성 (3시간)

  • ✓ Flutter 프로젝트 생성 및 4-layer 아키텍처 설계
  • ✓ Google Maps API 설정 (iOS/Android)
  • ✓ 개발 환경 테스트 (flutter run -d chrome)

1.2 Google Maps 기반 위치 서비스 (4시간)

  • ✓ Google Maps Widget 구현 (지도 표시, 줌/팬 제어)
  • ✓ GPS 위치 권한 요청 및 현재 위치 표시
  • ✓ 마커(Marker) 표시 및 위치 서비스 구현
  • ✓ 위치 서비스 오류 처리 (LocationError UI)

1.3 Naver Place API 데이터 연동 (3시간)

  • ✓ Naver Search API 통합 및 HTTP 클라이언트
  • ✓ 가게 정보 캐싱 (Hive 로컬 저장소)
  • ✓ Cafe 데이터 모델 설계 및 구현

1.4 검색 & 필터링 UI (2시간)

  • ✓ 검색 바(SearchBar) 구현 (텍스트 입력, 실시간 검색)
  • ✓ 필터 UI 구현 (거리, 평점 필터 선택)
  • ✓ 정렬 옵션 (거리순, 평점순 정렬)

1.5 가게 상세 페이지 (2시간)

  • ✓ 상세 정보 화면 UI 구현 (가게명, 주소, 평점, 리뷰 수)
  • ✓ 미니맵 통합 (GoogleMap 위젯 with 고정 줌 레벨)
  • ✓ 액션 버튼 (즐겨찾기, 길 찾기, 전화)

Phase 2: Firebase 인증 & 리뷰 시스템 ✓ 완료

기간: 10주차 (5월 26일) | 시간: 16시간 | 상태: 100% 완료

목표: 사용자 인증과 리뷰 시스템 구현 (완료)

2.1 Firebase 환경 설정 (1시간)

  • ✓ Firebase 패키지 활성화 (firebase_core, firebase_auth, cloud_firestore)
  • ✓ FirebaseService 생성 (Singleton 패턴)

2.2 도메인 계층 구현 (1시간)

  • ✓ User, Review 엔티티 정의
  • ✓ UserRepository, ReviewRepository 추상 인터페이스

2.3 데이터 계층 구현 (3시간)

  • ✓ FirebaseAuthDatasource (회원가입, 로그인, 로그아웃)
  • ✓ FirestoreUserDatasource (즐겨찾기 관리)
  • ✓ FirestoreReviewDatasource (리뷰 CRUD)
  • ✓ UserModel, ReviewModel (JSON 직렬화)
  • ✓ UserMapper, ReviewMapper

2.4 리포지토리 & Use Cases (2시간)

  • ✓ UserRepositoryImpl, ReviewRepositoryImpl
  • ✓ LoginUseCase, RegisterUseCase, LogoutUseCase
  • ✓ CreateReviewUseCase, GetReviewsForCafeUseCase, DeleteReviewUseCase

2.5 State Management (2시간)

  • ✓ AuthProvider (로그인, 회원가입, 즐겨찾기 상태 관리)
  • ✓ ReviewProvider (리뷰 로드, 작성, 삭제 상태 관리)
  • ✓ DI 컨테이너 업데이트

2.6 인증 UI 구현 (2시간)

  • ✓ LoginScreen (이메일/비밀번호 입력)
  • ✓ RegisterScreen (회원가입, 입력 검증)
  • ✓ main.dart 라우팅 (인증 상태 기반)

2.7 리뷰 UI 구현 (2시간)

  • ✓ ReviewCard (리뷰 표시, 별점, 삭제 버튼)
  • ✓ WriteReviewDialog (리뷰 작성 다이얼로그)
  • ✓ CafeReviewsSection (리뷰 섹션 통합)
  • ✓ CafeDetailScreen에 리뷰 섹션 추가

2.8 기능 구현 (Phase 3 예정)

  • □ 이미지 처리 및 업로드 (Firebase Storage)
  • □ 평점 평균 계산 및 분포 표시
  • □ 프로필 화면 및 편집 기능

3.7 UI/UX 폴란드 (2시간)

🔗의존성 관계

Phase 1 → Phase 2 → Phase 3

  • Phase 1의 70% 이상 완료 후 Phase 2 시작 가능
  • Phase 2는 Phase 1과 부분적으로 동시 진행 가능 (마지막 주에 통합)
  • Phase 3는 Phase 1, 2 완료 후 시작
  • 배포는 Phase 3 + 테스트 & 최적화 완료 필요

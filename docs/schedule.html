📌 10주차: Phase 1 시작 - 지도 & 기본 검색

주제: Flutter 프로젝트 초기화 및 Google Maps 통합
목표 달성도:
15%
항목 내용
목표 ✅ Flutter 프로젝트 구조 완성
✅ Google Maps 기본 기능 동작
✅ GPS 위치 권한 처리 완료
주요 작업 1. Flutter 프로젝트 생성 및 pubspec.yaml 설정
2. Google Maps API 키 발급 (Android/iOS)
3. Location 패키지 통합
4. 현재 위치 표시 기능
5. 에뮬레이터/실제 기기 테스트
산출물 • 완성된 Flutter 프로젝트 (GitHub 커밋)
• Google Maps 표시 화면 (스크린샷)
• 개발 환경 설정 가이드 (DEVELOPMENT.md)
검증 방법 ✓ 에뮬레이터에서 지도 렌더링 확인
✓ 현재 위치 권한 요청 & 표시
✓ 줌/팬 제어 정상 동작
✓ 콘솔 에러 없음
예상 시간 12-14시간
⚠️ 위험 요소 • iOS/Android API 키 설정 오류
• 권한 요청 안 됨 (특히 iOS)
→ 대응: 공식 문서 참고, 에뮬레이터 다시 시작

📌 11주차: Phase 1 계속 - Naver API & 검색

주제: 실제 가게 데이터 연동 및 검색/필터링 UI
기간: 2차 개발
항목 내용
목표 ✅ Naver Place API 연동 완료
✅ 지도에 가게 마커 500개+ 표시
✅ 검색 & 필터링 기능 동작
주요 작업 1. Naver Search API 연동
2. 가게 데이터 파싱 (Cafe, Restaurant 모델)
3. Hive를 이용한 로컬 캐싱
4. 검색 바 구현 (debouncing)
5. 필터 UI (거리, 평점, 카테고리)
6. 가게 상세 페이지 기본 구현
산출물 • 완성된 Naver API 통합 코드
• 검색 & 필터링 화면 (스크린샷)
• 가게 데이터 모델 (models/cafe.dart)
• API 통합 가이드 (API.md)
검증 방법 ✓ 지도에 500개+ 마커 표시
✓ 검색어로 결과 필터링
✓ 필터 적용 시 마커 업데이트
✓ 로컬 캐싱 작동 확인
✓ 오프라인 상태에서 캐시된 데이터 로드
예상 시간 12-14시간
⚠️ 위험 요소 • Naver API 일일 요청량 제한
• 마커 500개 이상 성능 저하
→ 대응: API 캐싱 강화, 마커 클러스터링

📌 12주차: Phase 2 시작 + 중간 발표

주제: Firebase 인증 및 사용자 시스템 구축
중요: 중간 발표 (12주차 말)
항목 내용
목표 ✅ Firebase 인증 시스템 완성
✅ 사용자 프로필 시스템 구현
✅ 리뷰 작성 기본 UI 완성
✅ 중간 발표 완료
주요 작업 1. Firebase 프로젝트 생성 & 설정
2. 이메일/Google OAuth 로그인
3. 사용자 프로필 Firestore 스키마 설계
4. 프로필 화면 UI 구현
5. 리뷰 데이터 모델 & Firestore 컬렉션
6. 중간 발표 준비 (슬라이드, 데모 영상)
산출물 • Firebase 설정 가이드
• 로그인/프로필 화면 (스크린샷)
• Firestore 스키마 다이어그램
• 중간 발표 슬라이드 (PPT)
• 현황 리포트 (개발 진도 ~70%)
예상 시간 14-16시간
⚠️ 위험 요소 • Firebase 인증 설정 복잡성
• 중간 발표 시간 압박
→ 대응: 일찍 시작, 발표 리허설

항목 내용
목표 ✅ 리뷰 작성/조회 완성
✅ 이미지 처리 및 캐싱 완성
✅ Phase 2 통합 테스트
주요 작업 1. 리뷰 작성 폼 완성
2. Firebase Storage 이미지 업로드
3. Cached Network Image 통합
4. 별점 위젯 구현
5. 리뷰 목록 표시 및 정렬
6. Phase 1, 2 통합 테스트
산출물 • 완성된 리뷰 시스템 코드
• 리뷰/이미지 처리 화면 (스크린샷)
• 성능 테스트 리포트
• 13주차 진행 현황 (개발 진도 ~90%)
예상 시간 14-16시간
⚠️ 위험 요소 • 이미지 업로드 성능 이슈
• Firebase Storage 비용 고려

📌 14-15주차: Phase 3 - 소셜 & 배포

주제: 소셜 기능 구현 및 앱 배포
중요: 최종 발표 (15주차 말)
항목 내용
목표 ✅ 팔로우/활동 피드 구현
✅ 리뷰 상호작용 (좋아요, 댓글)
✅ 성능 최적화 및 테스트
✅ Google Play/App Store 배포
✅ 최종 발표 완료
주요 작업 1. 팔로우/팔로워 시스템 구현
2. 실시간 활동 피드 (Firestore 리스너)
3. 좋아요/댓글 기능
4. 성능 프로파일링 및 최적화
5. 모든 기능 통합 테스트
6. Google Play Console & App Store 설정
7. 최종 발표 준비
산출물 • 완성된 앱 (모든 기능)
• Google Play 스토어 링크
• App Store 스토어 링크
• 최종 발표 슬라이드
• 사용자 가이드 (README)
• 기술 문서 및 API 가이드
예상 시간 20-24시간
⚠️ 위험 요소 • 배포 심사 기간 (1-3주)
• 대량 사용자 동시 접속 시 성능 저하
• 배포 후 버그 발견
→ 대응: 일찍 심사 신청, 부하 테스트, 배포 후 모니터링

  • 대량 마커 성능 저하

  • 배포 심사 거절

🟡 중간 위험

  • API 요청 한도 초과

  • 배포 문제: 앱 리뷰 정책 숙지, 모의 심사, 피드백 대응 계획
  • 일정 지연: 예비 시간 활용, 범위 축소 (선택사항 제거)
